# План разработки проекта: Нутрициолог Атакова Лада

## 📋 Обзор проекта

**Продукт:** Лендинг + PWA-приложение для нутрициолога  
**Стек:** GitHub Pages (хостинг) + Supabase (БД + Auth) + Продамус (оплата)  
**Домен:** dash.infinityfree.com  
**Целевая аудитория:** Клиенты нутрициолога, интересующиеся раздельным питанием

### ЭТАП 3: Настройка Supabase (2-3 дня)

#### 3.1. Создание таблиц базы данных

```sql
-- Пользователи (расширение auth.users)
profiles (
  id UUID REFERENCES auth.users,
  full_name TEXT,
  phone TEXT,
  email TEXT,
  created_at TIMESTAMP,
  consent_personal_data BOOLEAN,  -- согласие на обработку ПД
  consent_processing BOOLEAN,     -- согласие на обработку данных
  subscription_status TEXT,       -- free, trial, paid
  subscription_end TIMESTAMP
)

-- Отчеты о питании
reports (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES profiles(id),
  date DATE,
  time TIME,
  meal_type TEXT,               -- breakfast, lunch, snack, dinner, unplanned
  description TEXT,
  photo_url TEXT,               -- ссылка на фото в Storage
  comment TEXT,
  created_at TIMESTAMP
)

-- Замеры тела
measurements (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES profiles(id),
  date DATE,
  time TIME,
  weight DECIMAL,               -- вес в кг
  chest DECIMAL,                -- объем груди
  waist DECIMAL,                -- объем талии
  hips DECIMAL,                 -- объем бедер
  notes TEXT,
  created_at TIMESTAMP
)

-- Трекер воды
water_intake (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES profiles(id),
  date DATE,
  time TIME,
  amount_ml INTEGER,            -- количество мл
  created_at TIMESTAMP
)

-- Чат с куратором
messages (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES profiles(id),
  curator_id UUID REFERENCES profiles(id),
  content TEXT,
  sender_role TEXT,             -- client или curator
  created_at TIMESTAMP,
  is_read BOOLEAN
)

-- Подписки и оплаты
subscriptions (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES profiles(id),
  plan_type TEXT,               -- consultation, subscription, full
  status TEXT,                  -- pending, active, expired
  payment_id TEXT,              -- ID из Продамус
  start_date TIMESTAMP,
  end_date TIMESTAMP,
  created_at TIMESTAMP
)

-- Рекомендации куратора
recommendations (
  id UUID PRIMARY KEY,
  user_id UUID REFERENCES profiles(id),
  curator_id UUID REFERENCES profiles(id),
  content TEXT,
  created_at TIMESTAMP,
  is_read BOOLEAN
)
```

## ⚠️ Важные замечания

1. **152-ФЗ о персональных данных:** Обязательно получение явного согласия на обработку ПД, хранение данных на серверах в РФ (Supabase позволяет выбрать регион)

2. **Безопасность:** Все данные передаются только по HTTPS, чувствительные данные шифруются

3. **Оффлайн-режим:** PWA должен корректно работать без интернета, синхронизация при подключении

4. **Резервное копирование:** Настроить автоматический бэкап БД Supabase

5. **Масштабируемость:** Архитектура должна позволять добавление новых функций без полной переделки
